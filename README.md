## 🎯 项目简介

猫池短信系统是一款专为渠道操作用户设计的专业短信发送客户端软件。基于 Python + tkinter 开发，采用现代化的软件架构，支持多端口管理、任务调度执行、实时监控等核心功能，为短信发送业务提供稳定可靠的解决方案。

### 🏗️ 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                    猫池短信系统架构                           │
├─────────────────────────────────────────────────────────────┤
│  总管理后台 (PHP)    渠道管理后台 (PHP)    渠道操作端 (Python) │
│      ↓                    ↓                      ↓         │
│  ┌─────────┐         ┌─────────┐           ┌─────────┐      │
│  │总管理员 │ ──充值──→ │渠道管理员│ ──分配──→ │操作用户 │      │
│  │积分管理 │         │积分分配 │           │发送短信 │      │
│  └─────────┘         └─────────┘           └─────────┘      │
│                                                  ↓         │
│                                            ┌─────────┐      │
│                                            │猫池设备 │      │
│                                            │短信发送 │      │
│                                            └─────────┘      │
└─────────────────────────────────────────────────────────────┘
```

## ✨ 功能特性

### 🔐 用户认证与安全
- **多重验证**: 用户名 + 密码 + MAC地址绑定
- **设备授权**: 确保只有授权设备可以登录
- **会话管理**: 自动会话超时和安全退出
- **操作审计**: 完整的用户操作日志记录

### 📋 任务管理系统
- **智能任务创建**: 支持短信/彩信任务，灵活的内容编辑
- **精确进度控制**: 开始、暂停、恢复、停止任务执行
- **失败重试机制**: 智能重试失败消息，提高发送成功率
- **实时测试功能**: 单条测试验证，确保发送质量
- **批量号码导入**: 支持 Excel、CSV、TXT 多种格式
- **详细结果导出**: 成功/失败消息明细导出分析

### 🔌 端口管理中心
- **自动设备发现**: 智能扫描检测可用 COM 端口
- **实时状态监控**: 端口状态、运营商信息、发送统计
- **智能负载均衡**: 自动分配消息到空闲端口
- **灵活端口控制**: 批量启动、停止、配置管理
- **统计数据管理**: 发送统计清除和重置功能

### 📊 监控与统计
- **实时进度显示**: 任务执行进度实时更新
- **多维度统计**: 成功、失败、超时数量统计
- **监测提醒系统**: 发送指定条数后自动检测提醒
- **积分实时监控**: 自动积分余额监控和扣除

### 🎨 用户界面
- **现代化设计**: 橙色主题，简洁美观的界面
- **响应式布局**: 自适应窗口大小，优秀的用户体验
- **直观操作流程**: 三区域布局，操作逻辑清晰

## 🚀 快速开始

### 1. 环境准备

```bash
# 1. 克隆项目（或解压项目包）
git clone <repository-url>
cd catpool_sms_client

# 2. 检查 Python 版本
python --version  # 确保 >= 3.8

# 3. 创建虚拟环境（推荐）
python -m venv venv

# 4. 激活虚拟环境
# Windows
venv\Scripts\activate
# Linux/macOS  
source venv/bin/activate
```

### 2. 安装依赖

```bash
# 安装项目依赖
pip install -r requirements.txt

# 如果安装较慢，可使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple/
```

### 3. 数据库配置

```bash
# 1. 安装 PostgreSQL（如未安装）
# Windows: 下载官方安装包
# Ubuntu: sudo apt-get install postgresql postgresql-contrib
# macOS: brew install postgresql

# 2. 创建数据库
createdb sms_pool_system

# 3. 导入数据库结构
psql -d sms_pool_system -f jlmisplus.sql
```

### 4. 环境配置

```bash
# 1. 复制配置模板
cp .env.example .env

# 2. 编辑配置文件
# Windows
notepad .env
# Linux/macOS
nano .env
```

**重要配置项：**
```env
# 数据库连接
DB_HOST=localhost
DB_PORT=5432
DB_NAME=sms_pool_system
DB_USER=postgres
DB_PASSWORD=your_password

# 应用配置
APP_ENV=production
DEBUG=false
```

### 5. 项目完整性检查

```bash
# 运行项目检查工具
python project_check.py
```

### 6. 启动应用

```bash
# 启动应用
python app.py
```

## 📖 使用说明

### 登录系统

1. **启动程序**: 双击 `app.py` 或命令行运行
2. **输入凭据**: 输入操作用户名和密码
3. **设备验证**: 系统自动验证 MAC 地址绑定
4. **进入主界面**: 登录成功后显示用户积分和主操作界面

### 端口管理

#### 端口扫描和状态监控
```
┌─────────────────────────────────────┐
│ [全选][反选][选项][启动][停止]        │
│                                     │
│ COM1   中国移动   60    3    可用    │
│ COM2   中国联通   60   10    忙碌    │  
│ COM3   中国电信   60    0    离线    │
│                                     │
│ [清除全部记录][清除当前记录]         │
└─────────────────────────────────────┘
```

#### 端口操作步骤
1. **自动扫描**: 程序启动时自动检测可用端口
2. **查看状态**: 实时显示端口连接状态和统计信息
3. **选择端口**: 使用全选、反选或单独选择端口
4. **启动服务**: 点击"启动"激活选中端口
5. **监控运行**: 观察端口状态和发送统计

### 任务管理

#### 创建任务
1. **点击"添加任务"**: 打开任务创建对话框
2. **填写任务信息**:
   ```
   任务名称: [测试任务001]
   目标号码: [手动输入或文件上传]
   短信模板: [选择预设模板]
   号码模式: ○国内号码 ○国际号码
   主题: [彩信专用字段]
   短信内容: [编辑消息内容]
   ```
3. **验证信息**: 系统自动验证号码格式和内容
4. **保存任务**: 确认信息无误后创建任务

#### 任务执行控制
```
任务列表显示:
┌─────────────────────────────────────┐
│ 任务1    50%   100   5   运行中     │
│ [开始][暂停][测试][编辑]             │
│ [重试][导出完成][导出未完成][删除]   │
│                                     │
│ 任务2    25%    50   2   已暂停     │
│ [开始][暂停][测试][编辑]             │
│ [重试][导出完成][导出未完成][删除]   │
└─────────────────────────────────────┘
```

#### 任务操作说明
- **🟢 开始任务**: 启动任务执行，状态变更为"运行中"
- **⏸️ 暂停任务**: 暂停任务执行，可稍后恢复
- **🔄 重试失败**: 重新发送失败的消息
- **🧪 测试任务**: 发送测试短信验证配置
- **✏️ 修改内容**: 编辑任务名称和消息内容
- **📤 导出已完成**: 导出成功发送的消息明细
- **📤 导出未完成**: 导出失败/待发送的消息明细
- **🗑️ 删除任务**: 软删除任务（不会物理删除数据）

```
选项配置:

│
监测设置:
├── 发送成功: [1000] 条
└── 监测号码: [手机号码]
```

**配置说明**:
- **发送间隔**: 每条短信发送的时间间隔
- **卡片更换**: 发送多少条后需要清除记录
- **监测提醒**: 达到指定条数时发送监测短信

## 🛠️ 技术架构

### 架构设计原则
- **分层架构**: UI层 → 服务层 → 数据层，职责分离清晰
- **模块化设计**: 功能模块独立，便于维护和扩展
- **异步处理**: 避免界面阻塞，提升用户体验
- **错误处理**: 完善的异常捕获和恢复机制

### 技术栈详情

#### 前端界面 (UI Layer)
- **GUI框架**: Python tkinter (内置，无需额外安装)
- **界面设计**: 橙色主题，响应式布局
- **组件库**: 自定义组件库，模块化界面元素

#### 业务逻辑 (Service Layer)  
- **认证服务**: bcrypt密码加密 + MAC地址绑定
- **任务管理**: 多线程任务调度和状态管理
- **端口管理**: 串口通信和设备状态监控
- **积分系统**: 实时积分计算和扣除管理

#### 数据存储 (Data Layer)
- **数据库**: PostgreSQL + psycopg2
- **连接池**: 多线程安全的数据库连接池
- **事务管理**: 完整的事务处理和回滚机制

#### 核心组件 (Core Components)
- **串口通信**: pyserial 串口设备通信
- **文件处理**: pandas + openpyxl Excel/CSV处理  
- **系统监控**: psutil 系统资源监控
- **日志系统**: 多级别、多文件日志记录

### 数据库设计

#### 核心表结构
```sql
-- 用户体系 (3层架构)
super_admins        -- 总管理员表
channel_users       -- 渠道管理员表  
channel_operators   -- 渠道操作用户表

-- 积分管理

-- 业务核心

task_message_details   -- 消息发送明细表
```

#### 关键功能实现
- **积分控制**: 存储过程实现原子性积分操作
- **状态管理**: 任务和消息状态自动流转
- **数据完整性**: 外键约束和检查约束
- **性能优化**: 索引优化和查询优化

## 🚨 故障排除

### 常见问题及解决方案

#### 1. 程序无法启动
**症状**: 双击app.py无反应或报错
**解决方案**:
```bash
# 检查Python版本
python --version

# 检查依赖安装
pip list | grep psycopg2
pip list | grep pyserial

# 重新安装依赖
pip install -r requirements.txt

# 检查项目完整性
python project_check.py
```

#### 2. 数据库连接失败
**症状**: 登录时提示数据库连接错误
**解决方案**:
```bash
# 检查PostgreSQL服务状态
# Windows: 服务管理器中查看PostgreSQL服务
# Linux: sudo systemctl status postgresql

# 检查数据库配置
cat .env | grep DB_

# 测试数据库连接
psql -h localhost -U postgres -d sms_pool_system
```

#### 3. 端口扫描不到设备
**症状**: 端口管理区域显示为空
**解决方案**:
```bash
# 检查设备连接
# 1. 确认USB线连接正常
# 2. 检查设备电源状态
# 3. 查看设备管理器(Windows)或lsusb(Linux)

# 检查驱动安装
# Windows: 设备管理器 → 端口(COM和LPT)
# Linux: ls /dev/ttyUSB* 或 ls /dev/ttyACM*

# 检查端口权限(Linux)
sudo usermod -a -G dialout $USER
# 重新登录或重启生效
```

#### 4. MAC地址验证失败
**症状**: 登录时提示MAC地址未授权
**解决方案**:
```bash
# 查看当前MAC地址
# Windows: getmac
# Linux/macOS: ifconfig 或 ip link show

# 联系管理员绑定MAC地址
# 或临时禁用MAC验证(仅开发环境)
# .env文件中设置: SKIP_MAC_VERIFICATION=true
```

#### 5. 发送失败率高
**症状**: 短信发送成功率低
**解决方案**:
```bash
# 检查SIM卡状态
# 1. 确认SIM卡余额充足
# 2. 检查SIM卡信号强度
# 3. 验证运营商网络状态

# 调整发送参数
# 增加发送间隔: DEFAULT_SEND_INTERVAL=2000
# 减少并发任务: MAX_CONCURRENT_TASKS=1

# 检查内容格式
# 确认短信内容符合运营商规范
# 避免包含敏感词汇
```

#### 6. 积分扣除异常
**症状**: 积分余额显示不正确
**解决方案**:
```sql
-- 检查积分余额
SELECT operators_available_credits FROM channel_operators 
WHERE operators_username = 'your_username';
```
